PACKAGE STRUCTURE OF THE ABSTRACT SYNTAX TREE

 

The aim of the URIUM compiler's parsing phase is to read a source file and generate a data structure that stores all the information contained in that source file. This data structure is known as an abstract syntax tree. The classes used in the URIUM compiler to develop this data structure are found in the urium.ast package, which in turn includes three other subpackages. These packages are described below:

  • urium.ast: It is the root package for all classes used in the data structure. The package only contains the classes that develop the type system, which are used by many of the classes in the various subpackages.

  • urium.ast.expression: This package contains classes dedicated to describing the different arithmetic and logical expressions included in the URIUM language.

  • urium.ast.statement: It is the package that includes the classes that describe the different statements of the URIUM language: (if, while, endp, etc.).

  • urium.ast.struct: It is the package to which the classes that describe the high-level components of the URIUM language (libraries, procedures, etc.) belong.

 

THE urium.ast PACKAGE

 

The urium.ast package contains classes dedicated to describing the URIUM language's type system. The package consists of one interface and two classes.

  • urium.ast.Access: It is an interface that defines the constants used to describe the two types of access included in the language: public and private.

  • urium.ast.Type: It is a class that describes the data types included in the URIUM language. The class also defines the constants used to identify the types, as well as a set of predefined objects to represent these data types (int, char, boolean, out int, out char, out boolean).

  • urium.ast.TypeSystem: This is the class that develops the methods associated with the language's type system. The methods defined in this class are as follows:

    • boolean isInteger(DataType type): Checks that a data type is an integer.

    • boolean isBoolean(DataType type): Checks that a data type is a boolean.

    • boolean isChar(DataType type): Checks that a data type is a character.

    • boolean isNumeric(DataType type):Checks that a data type is numeric.

    • boolean isOrderable(DataType type1, DataType type2): Checks that two data types can be sorted with '>' or '<'.

    • boolean isComparable(DataType type1, DataType type2): Checks that two data types can be compared with '=' o '!='.

    • boolean isAssignable(DataType type1, DataType type2): Tests whether a value of data type 'type2' can be assigned to a variable of data type 'type1'.

    • boolean isReference(DataType type): Checks that a data type is a reference.

    • DataType getReference(DataType type): Gets the reference type corresponding to a basic type.

    • int getNumberOfBytes(DataType type): Gets the size (in bytes) of the data type.

     

 

THE  urium.ast.struct PACKAGE

 

The urium.ast.struct package contains classes dedicated to describing the high-level components (libraries, procedures) of the URIUM language. The package consists of four classes.

  • urium.ast.struct.LibraryDeclaration: It is the class that contains all the information collected from a source file; that is, it constitutes the root of the abstract syntax tree to be built. It consists of the following components:

    • name: This is the full name of the library, including the path to the library's source file. For example, the library described in the file 'urium/Console.ur' is described by the name "urium.Console".

    • imported: It is a hashtable with the names of the different libraries imported in the file and their aliases.

    • procedures: It is a vector that stores the list of descriptions of the procedures defined in the library.

    • isNative: It is a flag to indicate that it is a native library, that is, its object code has already been generated and should not be recompiled.

    The different methods of the class are limited to accessing and editing these fields.

  • urium.ast.struct.Procedure: Describes a procedure defined within a library. It contains the following fields::

    • libname: It is the name of the library to which the procedure belongs.

    • name: It is the name of the procedure.

    • access: It is the type of access to the procedure (public or private).

    • argument: Contains the list of arguments for the procedure call. Each argument is an object of the Variable class..

    • localVar: Contains the list of local variables defined in the procedure body. Each variable is an object of the Variable class.

    • body: Contains the procedure's statement block. It is an object of the urium.ast.statement.BlockStatement class..

    Most of the class's methods are dedicated to accessing and editing these fields. Listed below are some methods with different functionality.

    • int[] getArgumtentTypes(): Gets the list of argument types. Used to verify that no two procedures have the same name and the same argument types.

    • boolean match(String, int[]): Checks if the procedure corresponds to that name and those argument types.

  • urium.ast.struct.Variable: Contains the description of a local variable or a procedure argument. The fields of the class are as follows:

    • name: Variable name.

    • type: Data type of the variable.

  • urium.ast.struct.SymbolTable: This is the class that develops the compiler's symbol table. The symbol table contains the set of libraries analyzed by the compiler and is responsible for identifying the library and the procedure being analyzed at any given time. For this procedure, the symbol table constructs a stack of variable declaration scopes. When an instruction in the active procedure opens a new declaration scope (for example, a block of instructions in a while statement), a new scope is stacked in this class. Scopes are represented by hash tables that store variable names and references to the associated Variable objects. The methods defined in this class are as follows:

    • void addLibrary(LibraryDeclaration lib): Adds a library to the symbol table.

    • LibraryDeclaration getLibrary(String libname): Search the symbol table for a library with the given name.

    • LibraryDeclaration getActiveLibrary(): Gets the active library, that is, the library that is currently being analyzed.

    • void setActiveLibrary(String libname): Sets the active library.

    • LibraryDeclaration[] getLibraries(): Gets the list of libraries as an array.

    • Procedure getActiveProcedure(): Gets the active procedure, that is, the procedure that is currently being analyzed.

    • void setActiveProcedure(String name, int[] type): Sets the active procedure.

    • void createScope(): Creates a new scope (a hash table) and stores it at the top of the stack.

    • void deleteScope(): Removes the scope stored at the top of the stack.

    • void addLocalVariable(Variable var): Adds a variable declaration to the active procedure and stores it in the top-of-stack scope.

    • void addArgument(Variable var): Adds an argument to the active procedure and stores it in the top-of-stack scope.

    • Variable getVariable(String name): Searches for a variable with the given name in all stack scopes.

    • Variable getVariableInScope(String name): Searches for a variable with the given name in the top-of-stack scope.

 

THE  urium.ast.statement PACKAGE

 

The urium.ast.statement package contains classes that describe the various statements in the URIUM language. The package contents are as follows:

  • urium.ast.statement.Statement: This is an abstract class that serves as a superclass for any class that defines an statement. The class only contains the abstract method returns(), which must verify whether the statement always reaches an endp (end-procedure) or not (statements that always reach an endp are the endp statement itself, a block of statements that finish in an endp, and an if-else statement where both options always reach an endp).

  • urium.ast.statement.IfStatement: This is the class that describes an if statement. Its fields are as follows:

    • condition: Describes the condition of the if statement. It is a reference to an Expression object. It must be of boolean type.

    • thenInst: Describes the statement to execute when the condition is true. It is a reference to a Statement object.

    • elseInst: Describes the statement to execute when the condition is false. It is a reference to a Statement object. If the statement does not have an else clause, the field must have a null reference..

    The class's methods are limited to assigning and accessing the values ​​of these fields. The returns() method returns true if the thenInst and elseInst statements reach an endp..

  • urium.ast.statement.WhileStatement: This is the class that describes a while statement. It consists of the following fields:

    • condition: Describes the loop's control condition. It is a reference to a Expression object of boolean type.

    • body: Describes the body of the loop. It is a reference to a Statement object..

    The methods are limited to accessing and editing field values. The returns() method always returns false.

  • urium.ast.statement.EndpStatement: Describes an endp statement that indicates the end of the procedure. It does not contain any fields. The returns() method always returns true.

  • urium.ast.statement.AssignStatement: This is the class that describes an assignment statement. Its fields are:

    • lefthand: Contains the reference to the variable that appears on the left side of the assignment. It is an object of the Variable class.

    • expression: Represents the right-hand side of the assignment, that is, the expression that calculates the value to be assigned. It is a reference to an object of the Expression class. The type of the expression must match the type of the variable.

    The returns() method always returns false. The other methods simply retrieve and assign field values.

  • urium.ast.statement.CallStatement: Describes a procedure call statement. It consists of the following fields.

    •  procedure: Contains the reference to the procedure to call.

    • library: Contains the reference to the library to which the procedure belongs

    • call: This is the list of expressions to be used as arguments to the call.

  • urium.ast.statement.BlockStatement: It is a class that describes a block of statements. It is often used as the body of if and while statements. Its fields are:

    • list: Contains the list of statements in the block, that is, the list of Statement objects.

    • returns: It is a flag that indicates whether the statement block always reaches an endp. This flag is updated every time a new statement is added to the block..

    The most important method is addStatement(Statement), which adds a new statement to the block and updates the returns flag. The returns() method gets the value of the flag.

 

THE  urium.ast.expression PACKAGE

 

The urium.ast.expression package contains classes that describe the different types of expressions supported by the URIUM language. These expressions are used as the right-hand side of assignments, as arguments in procedure calls, and as conditions in if and while statements. The package contents are as follows:

  • urium.ast.expression.Expression: This is an abstract class that serves as a superclass for all classes that describe expressions. Its only field is type, which contains the data type of the expression.

  • urium.ast.expression.OperatorExpression: It is an abstract class that encapsulates all expressions that represent an operation on other expressions. This class is developed by the BinaryExpression and UnaryExpression subclasses..

  • urium.ast.expression.LiteralExpression: It is the abstract class that encapsulates all expressions that represent a literal value. This class is developed by the subclasses BooleanLiteralExpression, CharLiteralExpression, and IntegerLiteralExpression.

  • urium.ast.expression.ReferenceExpression: It is an abstract class that encapsulates all expressions that represent a reference to a variable.

  • urium.ast.expression.BinaryExpression: This class describes binary expressions, that is, expressions consisting of one operator and two operands. The class defines a series of constants to identify the different binary operators in the URIUM language: logical operators (AND and OR), arithmetic operators (PLUS, MINUS, PROD, DIV, and MOD), and comparison operators (EQ, NEQ, GT, GE, LT, and LE). The components of the class are as follows:

    • op: Binary operator code. Its value must be one of the constants defined in the class.

    • left: Left-hand operand of the expression. It is a reference to an Expression object.

    • right: The right-hand operand of the expression. It is a reference to an Expression object.

    The class's methods are the getters for these fields. The data type of the expression is passed in the constructor.

  • urium.ast.expression.UnaryExpression: This class describes unary expressions, that is, expressions consisting of an operator and a single operand. The class defines a series of constants to identify the different unary operators in the URIUM language: the logical negation operator (NOT) and the unary arithmetic operators (PLUS and MINUS). It also includes a constant (NONE) to indicate that no unary operator is applied to the operand. The components of the class are as follows:

    • op: Unary operator code. Its value must be one of the constants defined in the class.

    • exp: Operand of the expression. It is a reference to an Expression object.

    The class's methods are the getters for these fields. The data type of the expression is passed in the constructor.

  • urium.ast.expression.BooleanLiteralExpression: This class describes a boolean literal, that is, it represents the constants true and false. Its only field is value, which stores the literal's value, and its only method is this field's getter. The class contains two constructors that allow the literal to be constructed by parsing the lexeme (the strings "true" and "false") or directly from the value. The data type of the expression is boolean.

  • urium.ast.expression.CharLiteralExpression: This is the class dedicated to representing character literals. Its only field is value, which stores the value of the literal. The class contains two constructors: one based directly on the literal's value and another based on the lexeme. To parse the lexeme, the private method getCharFromString(String) is used, which analyzes the different ways of representing a character literal defined in the URIUM language: editable characters (for example, 'a'), non-editable characters described with the backslash (for example, '\n'), characters described in octal format (for example, '\071'), and characters described in Unicode format (for example, '\u007A'). The data type of the expression is char.

  • urium.ast.expression.IntegerLiteralExpression: Describes integer literals. Its only field is value, which stores the literal's value, and its only method is getValue(), which returns the value of that field. Constructors allow objects to be created from the lexeme (e.g., "325") or directly from the value. The data type of the expression is int.

  • urium.ast.expression.CallParameters: It is the only class in the package that is not a subclass of Expression. It describes the arguments to a procedure call, that is, the list of expressions used in the call definition. Its only field is parameter, which contains a list of expressions. The class's methods allow you to add expressions to the list ( addParameter() ), get the list ( getParameters() ), and get the data types of the expressions ( getTypes() ). The latter method is used to locate the procedure version by considering the data types of the call. (The URIUM language supports procedure overloading, that is, defining procedures with the same name that differ in the data types of their arguments.)

  • urium.ast.expression.VariableExpression: Describes an expression formed by referencing a local variable or a method argument. Its only field is var, which stores the reference to the Variable object that describes the variable. The data type of the expression corresponds to the data type of the variable.

  • urium.ast.expression.OutVariableExpression: Describes an expression formed by referencing a variable used as an output argument in a call. That is, the address of the variable is passed in the call so that the called procedure can modify the value of the referenced variable. Its only field is var, which stores the reference to the Variable object that describes the variable. The data type of the expression is a reference to the data type of the variable.
